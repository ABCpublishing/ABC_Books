# AWS Deployment Guide for ABC Books

This guide outlines the steps to deploy the ABC Books application (Express + MySQL) to Amazon Web Services (AWS) using a modern, scalable architecture.

## 🏗 Recommended Architecture
*   **Backend + Frontend (Single App)**: AWS App Runner
*   **Database**: AWS RDS (MySQL)
*   **Media Storage**: AWS S3 (Optional, for book covers)
*   **Security**: AWS Secrets Manager

---

## 1. Prepare Database (AWS RDS)
1.  Log in to the **AWS Management Console**.
2.  Go to **RDS** > **Create database**.
3.  Choose **MySQL**.
4.  Templates: **Free Tier** (to avoid costs).
5.  Settings: 
    *   DB instance identifier: `abc-books-db`
    *   Master username: `admin` (or your choice)
    *   Master password: `YourSecurePassword123`
6.  Connectivity: 
    *   **Public access**: Yes (temporarily, for migration).
    *   In the **VPC Security Group**, ensure Port **3306** is open to your IP address.
7.  Once created, copy the **Endpoint** (e.g., `abc-books.xyz.us-east-1.rds.amazonaws.com`).

### Migrate Data
Export your local database and import it to RDS:
```bash
# Export from local
mysqldump -u root -p abc_books > backup.sql

# Import to AWS
mysql -h your-rds-endpoint -u admin -p abc_books < backup.sql
```

## 2. Prepare Application (App Runner)
AWS App Runner provides an easy way to deploy from GitHub.

1.  Go to the **App Runner** console > **Create service**.
2.  Source: **Source code repository**.
3.  Connect your GitHub account and select `ABC_Books`.
4.  Deployment settings: **Automatic**.
5.  Build settings:
    *   Runtime: **Nodejs 18** (or 20)
    *   Build command: `cd backend && npm install`
    *   Start command: `cd backend && node server.js`
    *   Port: `3001`

---

## 3. Configuration (Environment Variables)
In App Runner settings, add the following Environment Variables. **Do not use a `.env` file on AWS.**

| Variable | Value |
| :--- | :--- |
| `DB_HOST` | *Your RDS Endpoint* |
| `DB_USER` | `admin` |
| `DB_PASSWORD` | *Your RDS Password* |
| `DB_NAME` | `abc_books` |
| `JWT_SECRET` | *A long random string* |
| `MAILGUN_API_KEY` | *Your Mailgun Key* |
| `MAILGUN_DOMAIN` | `mg.abcbooks.store` |
| `RAZORPAY_KEY_ID` | *Your Razorpay ID* |
| `RAZORPAY_KEY_SECRET` | *Your Razorpay Secret* |
| `FRONTEND_URL` | *The URL AWS provides after deployment* |

---

## 4. Domain & SSL
1.  App Runner automatically provides an `https://` URL.
2.  To use your custom domain (`abcbooks.store`):
    *   Go to **Custom domains** in the App Runner service.
    *   Add your domain and follow the instructions to add **CNAME** records in your domain registrar (e.g., Hostinger, GoDaddy).

---

## 5. Security Checklist
*   [ ] **VPC Security Groups**: Once migration is done, set RDS "Public Access" to **No**. Allow access only from the App Runner security group.
*   [ ] **Production Flags**: Ensure `NODE_ENV=production` is set in environment variables.
*   [ ] **CORS**: Ensure the `FRONTEND_URL` in environment variables matches your production domain.
